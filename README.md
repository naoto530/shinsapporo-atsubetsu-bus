# 六花亭周辺 バス検索PWA

新札幌駅バスターミナルから、六花亭周辺へ向かう対象路線を横断して検索できるスマートフォン向けWebアプリです。外部APIは使わず、時刻表は `data/timetable.json` で管理します。

## できること

- 現在日時から、平日 / 土曜 / 日曜・祝日を自動判定
- 現在時刻以降の便を、バス会社をまたいで発車時刻順に表示
- 発車時刻、あと何分、バス会社、路線名、乗り場、降車停留所を表示
- 乗車後、降車停留所までの標準所要時間と降車目安時刻を表示
- なーたんからの心の声メッセージ50個をランダム表示
- 任意の日付と時刻に変更して検索
- 最終便後は、翌日以降の始発を表示
- 平日、土曜、日曜・祝日の時刻表一覧を表示
- PWAとしてiPhone / Androidのホーム画面に追加可能
- オフライン時も最後に取得した時刻表を表示

## ローカルで起動

Service Workerを使うため、ファイルを直接開かずローカルサーバーで起動してください。

Windowsでは `start-preview.bat` をダブルクリックすると、プレビューサーバーが起動してブラウザが開きます。

```bash
python -m http.server 5173
```

起動後、ブラウザで次を開きます。

```text
http://localhost:5173
```

別のポートを使う場合は `5173` を任意の番号に変更してください。

## 時刻表データの更新

時刻表は `data/timetable.json` の `routes` にまとめています。

```json
{
  "id": "jr-ebetsu-n10",
  "operator": "JR北海道バス",
  "routeName": "江別方面行（新24・新26・新27・新29ほか）",
  "lane": "北レーン10番",
  "alightingStop": "厚別東4条4丁目",
  "travelTimeMinutes": 5,
  "timetables": {
    "weekday": ["06:38", "06:42", "07:01"],
    "saturday": ["06:41", "06:56", "07:01"],
    "holiday": ["06:41", "06:56", "07:01"]
  }
}
```

更新時は次の流れがおすすめです。

1. 各バス会社の公式時刻表を確認する
2. `weekday`、`saturday`、`holiday` の時刻を差し替える
3. 降車停留所までの所要時間が変わる場合は `travelTimeMinutes` を更新する
4. `metadata.updatedAt` と `metadata.version` を更新する
5. Vercelへ再公開する

アプリはオンライン時に `data/timetable.json` を毎回取り直します。再公開後、家族の端末ではページを開くか「時刻表を更新」を押すと最新版が反映されます。オフライン時は最後に取得できた時刻表を使います。

## 入力済みデータの出典

- JR北海道バス: No.10 時刻表 2026年4月1日改正
- 夕鉄バス: 江別版 時刻表 2025年7月1日改正
- 北海道中央バス: 新さっぽろ駅 白35 / 白38 時刻表 2026年4月1日改定

所要時間は停留所間の標準時刻をもとにした目安です。道路状況、積雪、イベント、遅延により実際の降車時刻は前後します。

## Vercelで公開

ビルド不要の静的サイトなので、Vercelにそのまま配置できます。

1. このフォルダをGitHubなどのリポジトリに入れる
2. Vercelで「Add New Project」を選び、リポジトリをImportする
3. Framework Presetは「Other」を選ぶ
4. Build Commandは空欄、Output Directoryも空欄または `.` にする
5. Deployする

公開URLを家族に共有すれば、インストール不要でブラウザから利用できます。

## ダイヤ改正の定期チェック

GitHubに配置すると、`.github/workflows/check-timetable.yml` により公式データの定期チェックを実行できます。

- 通常期: 週1回チェック
- ダイヤ改正が多い時期: 3月、4月、9月、10月は毎日チェック
- 手動実行: GitHub Actionsの `Check timetable sources` から `Run workflow`

チェック内容は `scripts/check_timetable_updates.py` にまとめています。

- 中央バス 白35 / 白38: GTFSから対象停留所の時刻を抽出し、変化があれば確認用Pull Requestを作成
- 夕鉄バス: GTFSとPDF時刻表の便数差があるため、現時点では変更検知と確認メモのみ
- JR北海道バス: 公式PDFの変更を検知し、変更時は手動確認

Pull Requestを確認して問題なければマージします。VercelをGitHub連携していれば、マージ後に自動で再公開され、家族の端末にも最新版の `data/timetable.json` が反映されます。

自動作成されるPull Requestは、GitHubユーザー `naoto530` を担当者・レビュアーに指定します。GitHubの通知設定やスマホアプリ通知を有効にしておくと、ダイヤ変更候補に気づきやすくなります。

## ホーム画面に追加

iPhoneではSafariで公開URLを開き、共有ボタンから「ホーム画面に追加」を選びます。AndroidではChromeで公開URLを開き、メニューから「ホーム画面に追加」または「アプリをインストール」を選びます。

## 注意

同梱の時刻表は、2026年7月21日に各社公式時刻表を確認して入力した実データ版です。ダイヤ改正、臨時便、運休などで変わる可能性があるため、公開前や実際の利用前には各社公式情報も確認してください。
